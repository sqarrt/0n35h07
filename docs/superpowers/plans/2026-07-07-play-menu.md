# play-menu Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Экран «Играть» без вкладок: карусель Duel/Battle/War, единые слоты (инвайт/бот/код в каждом режиме, пер-слотовая сложность бота), кнопка-автомат ПОИСК/СТОП/READY, сцена меню на 4 спота-квадрата с анимацией появления (+закрытие ballArt-бага лобби-задника).

**Architecture:** `RoomSession` почти не меняется (только пер-слотовая сложность). UI пересобирается: `LobbyTabs`/`ModePicker` умирают, `LobbySeats`+`LobbySeatsGrid` сливаются в один `Seats` с раскладками по режиму, новый `ModeCarousel`, `Lobby.tsx` = 3 секции + кнопка-автомат. Веб-флоу упрощается: хост создаёт комнату с кодом, гость джойнится полем «по коду» (двухфазная жеребьёвка `enterRoomNegotiated` и веб-`DualMatchmaker` умирают). Duel-ПОИСК (desktop) остаётся на `SteamQuickMatch`; в Battle/War кнопки ПОИСК нет до ветки backfill. `MenuBackdrop` получает 4 спота, позу `lobby4` и единую анимацию появления.

**Tech Stack:** React/TS, Vitest + testing-library, Playwright; **frontend-design skill** на задачах 3, 4, 6 (карусель, сиденья, анимация появления).

**Спека:** `docs/superpowers/specs/2026-07-07-play-menu-design.md` §2–5, ветка 1 §7.

## Global Constraints

- Ветка `feature/play-menu` от `release_1.1.0`; не пушить; мерж после одобрения.
- Полный `npm run test` — только в финальном чекпоинте после подтверждения.
- Имена режимов Duel/Battle/War — константы, НЕ локализуются; новые подписи UI — ключи в `Dict` + все 10 локалей.
- UI не должен «прыгать»: фиксированные высоты секций, кнопка-автомат одного размера во всех состояниях (правило CLAUDE.md).
- Steam-рантайм в тестах недоступен: Duel-квик-матч не e2e-ится (как и раньше), инвайт-флоу — компонентно.
- Все правки e2e-хелперов централизованно (`tests/helpers.ts` + локальные хелперы спеков) — геймплейные спеки не должны знать деталей лобби.

### Task 4: Единый Seats v2 (frontend-design)

**Files:** Rewrite `src/components/lobby/LobbySeatsGrid.tsx` → `src/components/lobby/Seats.tsx` (заменяет и `LobbySeats.tsx`; старые удаляются в Task 5), Modify `src/components/lobby/types.ts`, Test Create `tests/unit/Seats.test.tsx` (поглощает LobbySeatsGrid.test.tsx).

**Produces:** `<Seats mode isHost seats onSeatClick(slot,zone) onBotRemove onBotName(slot,name) onBotDifficulty(slot,d) invite={pending,onInvite(slot),onCancel(id)} joinCode={code|null} searching />`

Раскладки: `'1v1'` — ряд из 2 с VS; `'2v2'` — 2 столбца×2 с VS и рамками TEAM_COLORS; `'ffa'` — столбец из 4 по центру.

Состояния сиденья (единый рендер для всех режимов):
- **Пустое (хост, не searching)**: ДВЕ пунктирные зоны: `seat-invite-<slot>` («＋ Пригласить друга»; на вебе по клику раскрывает `joinCode` с копированием — существующий код-блок) и `seat-addbot-<slot>` («Добавить бота», текст без эмодзи).
- **Пустое (гость)**: клик по сиденью = пересесть (`onSeatClick(slot,'move')`).
- **Бот (хост)**: имя-инпут (`lobby-bot-name-<slot>`, клик по сиденью — реролл), ✕ `lobby-bot-remove-<slot>`, сегмент normal/passive ВНУТРИ сиденья (`seat-diff-<slot>-<d>`).
- **Человек**: имя/цвет/ready-подсветка (как сейчас).
- **Ожидание инвайта**: из steam-lobby4 (имя+пульс+✕ `lobby-invite-cancel-<slot>`).
- **searching**: свободные сиденья крутят имена (существующий spin), зоны скрыты.

- [ ] **Использовать frontend-design skill** (состояния сидений, пунктирные зоны, фикс-высоты).
- [ ] Компонентные тесты (расширение существующих grid-тестов): обе зоны на пустом хостовом; клик-зоны зовут правильные хендлеры; per-slot difficulty сегмент; searching скрывает зоны и крутит спин; веб-режим (joinCode) раскрывает код.
