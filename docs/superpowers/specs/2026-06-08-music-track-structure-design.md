# Музыкальная форма фонового трека матча — дизайн

Дата: 2026-06-08 · Ветка: `feat/audio` · Статус: согласовано, готово к плану

## Проблема

Текущая музыка матча (см. `2026-06-07-game-music-playback-design.md`) собирается так: на каждую роль
(`kicks/bass/lead/sfx`) раз в `SWAP_EVERY_LOOPS = 4` лупа (32с) детерминированно выбирается **один**
стем, и так до конца матча. На слух это дало две проблемы:

1. **Однообразие** — нет драматургии: плотность и состав почти не меняются, только «перемешиваются»
   стемы. Нет ощущения сочинённого трека.
2. **«Каша» от текучки** — стем (особенно кик: их 32 в библиотеке) почти всегда подменяется на новый
   раз в 32с, грув не успевает закрепиться в узнаваемый.

## Цель

Заложить в алгоритм **детерминированную песенную форму** — арку «интро → тело → аутро», которая:
- всегда **начинается интро и заканчивается аутро** (а не зацикливается);
- **держит грув стабильным внутри секции** (лечит кашу) и меняет состав/плотность **на границах
  секций** (даёт драматургию);
- **узнаваема** (повторяющиеся мотивы) и при этом **вариативна** (лёгкая вариация при повторах),
  чтобы длинный матч не звучал дословно одинаково;
- детерминирована от сида матча → одинакова у обоих игроков (как и сейчас).

## Подход: арка, а не петля

Трек — это арка с гарантированными краями, а не цикл. Тип секции вычисляется чисто из
`(loopIndex, remainingMs)`:

- **Интро** — пока `loopIndex < INTRO_LOOPS` (начало матча, по счётчику лупов движка).
- **Аутро** — когда `remainingMs ≤ OUTRO_MS` (конец матча по остатку времени). Надёжно, потому что
  **матч завершается только по таймеру** (3/5/10 мин; досрочного конца по счёту нет — проверено в
  `Match.endMatch`, причины только `time | disconnect`).
- **Тело** — всё между ними: фиксированный паттерн секций (куплет/припев/бридж/соло), повторяется с
  лёгкой вариацией ровно столько, сколько нужно, чтобы добить хронометраж.

**Геймплей больше не влияет на музыку.** Привязка к первому убийству (`firstBloodDone`) убирается —
интро теперь чисто музыкальное (начало матча). Единственная связь с матчем — остаток времени для
аутро (уже отслеживается как `lastRemainingMs`).

## Каталог секций

| Секция | Роли | Длина | Идея |
|---|---|---|---|
| Интро  | kicks + bass                    | `INTRO_LOOPS` (4 лупа, 32с) | завести грув |
| Куплет | kicks + bass + sfx + **лид-куплета** | 4 лупа | основной грув, свой лид |
| Припев | kicks + bass + sfx + **лид-припева** | 4 лупа | кульминация фразы, лид-хук |
| Бридж  | bass + sfx (кики убираем)       | 2 лупа (16с) | breakdown, спад напряжения |
| Соло   | kicks + **лид-соло**            | 4 лупа | лид на переднем плане, разреженный ритм |
| Аутро  | kicks + **лид-припева**         | по остатку (`OUTRO_MS` ≈ 16с) | возврат хука, завершение |

Решения по запросу владельца:
- **У куплета свой лид** (отличный от лида припева) — секции узнаются по разным лид-мотивам.
- **Аутро использует лид припева** — главный хук возвращается в конце (целостность формы).

**Паттерн тела** (зацикливается, но с вариацией стемов на каждом проходе):

```
Куплет → Припев → Куплет → Припев → Бридж → Соло → Припев →  (повтор)
```

≈ 26 лупов ≈ 3.5 мин на проход. Для матча 5 мин — ~1.2 прохода, 10 мин — ~2.5 прохода; повторные
проходы используют другие варианты стемов (см. вариацию).

## Детерминизм и входы

`compose(seed, loopIndex, library, remainingMs) → Arrangement` — **единственное место правил**
(чистая функция). Тип секции и состав считаются внутри:

1. `remainingMs ≤ OUTRO_MS` → аутро.
2. иначе `loopIndex < INTRO_LOOPS` → интро.
3. иначе тело: `bodyLoop = loopIndex - INTRO_LOOPS`; проход по паттерну тела с накоплением длин
   секций → определяем текущий **тип секции**, её **порядковый номер появления** (для вариации) и
   позицию лупа внутри секции (для орнамента).

Всё — чистые функции от `loopIndex` (общий у пиров благодаря синхронному старту ритуала) и `seed`
(общий из кода лобби). `remainingMs` хост-авторитетно синхронизируется (как сейчас для finale).
Семпл-синхрона между машинами по-прежнему нет (и не нужно) — совпадает **последовательность**
аранжировок.

## Стемы: узнаваемость + вариативность

Стем выбирается по ключу и **держится стабильно внутри секции** (никакой подмены внутри — это и
лечит кашу). Смена — только на границе секции, через уже существующий кроссфейд движка.

Две группы ролей (продиктовано библиотекой: bass 7, kicks 32, lead 11, sfx 9 — бас самый узкий):

- **Фундамент — `bass`, `kicks`.** Ключ `(seed, role, foundationVariant)`, **не** зависит от типа
  секции → бас/кик единые по матчу (узнаваемый фундамент). `foundationVariant` меняется медленно —
  раз в полный проход паттерна тела (пул `FOUNDATION_POOL = 2`). Если секция роль не использует (бридж
  без кика) — голос просто отсутствует; но когда кик/бас звучат, это тот же выбранный вариант.
- **Цвет — `lead`, `sfx`.** Ключ `(seed, role, sectionType, colorVariant)` → характер секции. Лиды
  разных секций различны автоматически (разные `sectionType`). `colorVariant = (номер появления
  секции) mod COLOR_POOL` (`COLOR_POOL = 3`, ограничивается размером библиотеки роли) → припев №1 и
  №3 одинаковы, №2 другой: узнаваемо, но не дословный повтор.

Выбор индекса — `mulberry32` от смешанного ключа (как в текущем `pickVoice`: соль роли, соль типа
секции, золотое сечение для варианта). Инвариант стабильности жеребьёвки сохраняется: отдельные
потоки на каждое решение, добавление/убирание роли не сдвигает выбор остальных.

**Аутро-лид** = `pickStem(seed, 'lead', 'chorus', 0)` — фиксированный основной вариант лида припева.

## Орнамент: смешивание лидов на один луп

На детерминированно выбранных **одиночных** лупах поверх кладётся **второй лид всего на один луп** —
короткая «перекличка» лид-на-лид, без долгой мути. Правило v1: на **последнем лупе** секций припев
и соло добавляется второй лид-голос = лид **другой** секции (для припева — лид куплета, для соло —
лид припева) с пониженным гейном (`ORNAMENT_GAIN ≈ 0.5`), только на этот луп.

Реализация бесплатна для движка: орнаментный голос присутствует в аранжировке этого лупа и
отсутствует в следующем → движок сам вводит его кроссфейдом-ин и выводит хвостом-аут на краях лупа.

## Константы (стартовые значения, тюнятся в MusicDirector — единственное место правил)

```
INTRO_LOOPS       = 4         // длина интро в лупах
OUTRO_MS          = 16_000    // последние N мс — аутро
SECTION_LOOPS     = { verse:4, chorus:4, bridge:2, solo:4 }
BODY_PATTERN      = [verse, chorus, verse, chorus, bridge, solo, chorus]
FOUNDATION_POOL   = 2         // вариантов bass/kicks (ротация раз в проход тела)
COLOR_POOL        = 3         // вариантов lead/sfx на тип секции (clamp по размеру библиотеки)
ORNAMENT_GAIN     = 0.5       // гейн второго лида на одно-луповом орнаменте
```

(Гейны ролей берём из текущего `ROLE_GAIN`; per-section гейн-профили — возможное будущее тюнингование.)

## Вне области

- Реактивность на геймплей (накал/серии убийств) — намеренно убрана (владелец выбрал чистую форму +
  аутро по таймеру).
- Сэмпл-точная синхронизация между машинами — не нужна (совпадает последовательность).
- Музыка в меню и SFX — отдельные задачи (см. `ONESHOT_GAME_BRIEF.md` / память `audio-next-steps`).
- Возможное расширение библиотеки баса (сейчас 7) ради большего разнообразия фундамента — опционально,
  заказ соседнему редактору.
