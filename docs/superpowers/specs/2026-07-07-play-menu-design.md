# Экран «Играть»: карусель режимов, единые слоты, докомплектация

Дата: 2026-07-07. Статус: дизайн согласован. Базируется на закрытой спеке
`2026-07-07-multiplayer-4p-mesh-design.md` (меш, слоты, режимы — всё в release_1.1.0).

## 1. Цель

Убрать вкладки (matchmaking / friend / bot) и пересечение «режим × способ сбора».
Один экран: выбираешь РЕЖИМ, собираешь состав ЛЮБЫМ способом (друзья, боты,
матчмейкинг-докомплектация) в одних и тех же слотах, жмёшь READY.

Имена режимов едины во всех локалях: **Duel** (1v1), **Battle** (2v2), **War** (FFA).

## 2. Каркас экрана (сверху вниз)

1. **Карусель режимов**: три плитки — выбранная в центре, обе соседние видны по
   бокам, меньше и полупрозрачные; переключение стрелками/кликом по соседней,
   с циклическим переходом. Смена режима = `RoomSession.setMode` (существующий
   гейт «занятых больше лимита — блок»). Гость видит карусель заблокированной
   (режим владеет хост).
2. **Карта и время** — существующие `MapPicker`/`TimePicker`, лок при поиске /
   у гостя как сейчас.
3. **Кнопка-автомат** (одна):
   - свободные слоты, поиск не идёт → **ПОИСК** (на вебе скрыта);
   - поиск идёт → **СТОП** + индикация (см. слоты);
   - состав полный ИЛИ я гость → **READY** (существующая семантика:
     все занятые ready → старт; переименований нет);
   - я ready → «ждём остальных».
   - Гейты: ожидается инвайт → ПОИСК недоступен; идёт поиск → «пригласить» и
     «добавить бота» недоступны.

Вкладок больше нет; «тренировка с ботом» = Duel + «Добавить бота».

## 3. Слоты

Раскладки: **Duel** — 2 сиденья в ряд с VS; **Battle** — 2 столбца × 2
(левый = команда A, правый = B, рамки `TEAM_COLORS`) с VS; **War** — 1 столбец
из 4 по центру.

Состояния сиденья:
- **Пустое (хост)**: две кликабельные зоны с пунктирной границей —
  «＋ Пригласить друга» и «Добавить бота» (текст, без эмодзи).
- **Пустое (гость)**: клик = пересесть (существующий `setSlot`).
- **Бот (хост)**: имя-инпут (клик по сиденью — реролл имени), ✕ — освободить,
  **сегмент-пикер normal/passive внутри сиденья** — сложность становится
  ПЕР-СЛОТОВОЙ (`setBotDifficulty(d, slot)`; сейчас красит всех ботов).
- **Человек**: имя, личный цвет, ready-подсветка — как сейчас.
- **Ожидание инвайта**: имя друга + пульс + ✕ (ветка steam-lobby4, без изменений).
- **Во время поиска**: свободные сиденья крутят случайные имена (существующий spin).

## 4. Веб-путь

Тот же экран. «Поиск» скрыт (матчмейкинг Steam-only). «Пригласить друга» на
пустом сиденье показывает **код комнаты** (генератор + копирование — существующие);
внизу экрана постоянное компактное поле **«Присоединиться по коду»** (замена
старого RoomCodeField-флоу): ввёл код → вошёл гостем, режим/слоты приезжают из
Assign. Веб получает 2v2/FFA с друзьями по коду.

## 5. Сцена MenuBackdrop

- **4 спота образуют квадрат**: слоты 0 (хост) и 1 — одна диагональ, 2 и 3 —
  другая. Новая камерная поза «lobby4» в `menuCameraPoses.json` (Duel может
  остаться на текущей).
- Шары — по занятым слотам из ростера, с настоящей парой цветов, моделью и
  **ballArt** (`computeBalls`/`specOf` переписываются и перестают терять арт —
  закрывает отложенный баг «рисунок пропадает в лобби»).
- **Анимация появления** при занятии слота (человек или бот): новая, единая для
  всех (не зависит от их respawn-стиля) — материализация на споте (~0.5–0.7 с,
  масштаб с овершутом + вспышка; хореография — frontend-design на исполнении).
  Освобождение слота — быстрое зеркальное растворение.
- Во время поиска сцена показывает только реальных участников (спин — в UI).

## 6. Матчмейкер докомплектации (Steam)

Обобщение квик-матча: поиск заполняет ПУСТЫЕ сиденья твоей комнаты чужими
одиночками и/или сажает тебя-одиночку в чужую недоукомплектованную комнату.

- **Метаданные лобби** (Steam lobby data): `mode`, выбор карты, выбор времени,
  версия протокола. Лимит членов — по режиму (Duel=2, иначе 4; новая
  Rust-команда set-member-limit).
- **Хост жмёт ПОИСК**: лобби private→public (Rust-команда set-lobby-type),
  ждёт джойнов; СТОП → обратно private. Друзья и боты в составе остаются —
  заполняются только пустые сиденья.
- **Одиночка**: сканирует публичные лобби (фильтры: mode, версия, совместимость
  карты/времени) → джойнится в самое заполненное → обычный HELLO сажает в слот;
  подходящих нет → публикует своё лобби (симметрия текущего try-join-else-host).
- **Слияние групп НЕ делаем (v1)**: комната с ≥2 людьми только публикуется и
  ждёт одиночек; два неполных дуэта друг друга не найдут — осознанное
  ограничение (редкий кейс, боты — всегда запасной вариант).
- **Гонки**: лобби заполнилось между сканом и джойном → отказ по лимиту /
  `room full` на HELLO → сканируем следующее. Карта/время с чужими —
  существующий `resolveMatchParams` при HELLO.
- Старт: полный состав + все READY (докомплектованные жмут READY как все).

## 7. Ветки и порядок

1. **`feature/play-menu`** — секции 2–5 целиком; «Поиск» работает только в Duel
   через существующий SteamQuickMatch; в Battle/War кнопки ПОИСК до ветки 2 просто
   НЕТ (кнопка-автомат сразу в READY-логике — состав собирается друзьями/ботами).
   Играбельно сразу.
2. **`feature/backfill-matchmaking`** — секция 6: Rust-команды (set-lobby-type,
   set-member-limit, lobby data get/set, фильтрованный список), BackfillMatchmaker
   на TS, включение ПОИСКА во всех режимах.

## 8. Тесты

- **play-menu**: юниты — пер-слотовая сложность (RoomSession), компонентные —
  карусель/сиденья/кнопка-автомат, computeBalls (4 спота, ballArt); e2e web —
  карусель, джойн по коду в Battle/War, боты, старт. Спеки lobby-tabs/room
  переписываются (вкладок нет).
- **backfill**: cargo check; BackfillMatchmaker юнитами через фейковый
  Steam-интерфейс; e2e нет (нет Steam-рантайма) — обязательный живой смоук на
  двух аккаунтах до релиза.

## 9. Осознанные компромиссы

- Слияние неполных групп в матчмейкинге — v2.
- Квик-матч-«пары» как отдельный примитив умирает — его заменяет докомплектация
  (Duel-поиск = та же докомплектация с лимитом 2) в ветке 2; до неё Duel-поиск
  живёт на старом SteamQuickMatch.
- Веб без поиска (как и сейчас); веб-код-флоу теряет «оба вводят код» — теперь
  хост создаёт и показывает код, гость вводит (роль детерминирована, двухфазная
  жеребьёвка ролей уходит в прошлое вместе со старым флоу — упрощение).
- Визуальная полировка (карусель, анимация появления) — frontend-design в ходе
  исполнения ветки 1.
