# Редактор карт: автосохранение

Дата: 2026-07-08 · Ветка: `feature/editor-copy-paste` (продолжение работ по редактору, от `release_1.1.0`)

## Цель

Редактор карт сохраняет работу сам: пауза в правках — карта уже на диске. Ручная кнопка SAVE
остаётся как принудительный немедленный сейв. Случайные разрушения страхуются бэкапом состояния
на начало сессии с кнопкой REVERT.

## Поведение

### Автосейв

- Триггер: **дебаунс 3 с** (`AUTOSAVE_DEBOUNCE_MS = 3000`) после последнего изменения данных
  карты: воксели, `half`, `floorColor`, `wallColor`, `spawns`, `showGridInGame`.
- Начальная загрузка карты и загрузка бэкапа (REVERT) автосейв **не** триггерят: данные,
  пришедшие с диска, помечаются как «чистые».
- Каждый сейв пишет **все три артефакта**, как текущий SAVE: `raw.json` + `geo.json` +
  `preview.png` (offscreen-рендер через `ThumbnailRenderer`).
- Сейвы сериализуются: пока идёт запись, новые изменения коалесцируются в один повторный сейв
  после завершения текущего (двух параллельных `ThumbnailRenderer` не бывает).
- Статус в боковой панели: `saving…` → `saved HH:MM:SS` → (при ошибке) `save error`.
- Кнопка SAVE остаётся: немедленный `save()` без ожидания дебаунса.

### Бэкап и REVERT

- В допустимые файлы моста добавляется `backup.json` (на список карт не влияет — он фильтрует
  папки по `raw.json`).
- При открытии **существующей** карты редактор один раз пишет загруженное состояние в
  `backup.json` — «состояние на начало сессии». Для новой карты бэкапа нет.
- Кнопка **REVERT** в боковой панели видна только при существующем бэкапе; по клику —
  `confirm()` → загрузка бэкапа в редактор (как `loadInto`); дальше автосейв сам перезапишет
  raw/geo/preview.
- Бэкап перезаписывается при каждом новом открытии карты (сессии редактора).

### Закрытие вкладки

- На `pagehide`, если есть несохранённые изменения, шлётся **keepalive**-запись `raw.json` +
  `geo.json` (превью на выгрузке не отрендерить — догонится следующим сейвом в новой сессии).

## Архитектура

### Мост (`build/vite-plugins/editorMaps.ts`)

`PARTS` + `CT` дополняются `backup.json` (`application/json`). Другие изменения моста не нужны
(GET/PUT уже универсальны по `part`).

### API (`src/editor/mapsApi.ts`)

- `loadBackup(id): Promise<MapData | null>` и `saveBackup(id, map): Promise<boolean>` — по
  образу `loadMap`/`saveMap`.
- `put(...)` получает опциональный `keepalive` (проброс в `fetch`) для записи на `pagehide`.

### Хук `useMapSaver` (`src/editor/useMapSaver.ts`)

Владеет конвейером сохранения, который сейчас размазан по `doSave`/`onThumb`/`thumbMap` в
`MapEditor.tsx`:

```ts
function useMapSaver(name: string): {
  save: (data: MapData) => void        // поставить сейв в очередь (коалесцируется, если идёт запись)
  flush: (data: MapData) => void       // keepalive raw+geo для pagehide
  status: string                       // '' | 'saving…' | 'saved HH:MM:SS' | 'save error'
  thumbEl: ReactNode                   // ThumbnailRenderer текущего сейва (рендерится в MapEditor)
}
```

Внутри: очередь глубиной 1 (`busy` + `queued: MapData | null`), запись raw → geo → монтирование
`ThumbnailRenderer` → `saveThumbnail` → статус/время; если за время записи пришёл новый `save` —
повтор с последними данными.

### `MapEditor.tsx`

- `doSave`/`onThumb`/`thumbMap`/`status` заменяются на `useMapSaver`; SAVE-кнопка зовёт
  `save(buildMap())`.
- Автосейв-эффект: по изменению зависимостей данных, при `loaded` и «грязном» состоянии —
  таймер `AUTOSAVE_DEBOUNCE_MS`, затем `save(buildMap())`. Флаг «чистых» данных выставляется
  в `loadInto` (начальная загрузка и REVERT).
- Бэкап: в эффекте загрузки после успешного `loadMap` — `saveBackup(name, data)`,
  `setHasBackup(true)`.
- REVERT: кнопка рядом с SAVE, видимая при `hasBackup`; `confirm()` → `loadBackup` → `loadInto`.
- `pagehide`-слушатель: при несохранённых изменениях `flush(buildMap())`.

## Вне объёма

- Undo/redo, версионирование бэкапов (бэкап один, на начало сессии).
- Тумблер вкл/выкл автосейва.
- Серверная компиляция geo.

## Тесты

Новой чистой логики нет — вся фича — склейка React/сеть/офскрин-рендер, которая по проектному
паттерну юнитами в jsdom не покрывается (нет WebGL и dev-моста). Новые юнит-тесты не добавляются;
существующие (`editorStore`, `editorSelection`) не затрагиваются. Проверка — руками в dev:
автосейв по паузе, коалесценция при непрерывной стройке, REVERT, keepalive при закрытии вкладки.

## Процесс

- Работа продолжается в `feature/editor-copy-paste` (обе фичи редактора едут одним мержем в
  `release_1.1.0` после аппрува).
- Тесты запускает пользователь; агент — `npx tsc -b --noEmit` и `npm run lint`.
