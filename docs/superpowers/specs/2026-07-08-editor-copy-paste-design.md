# Редактор карт: выделение, копирование и вставка блоков

Дата: 2026-07-08 · Ветка: `feature/editor-copy-paste` (от `release_1.1.0`)

## Цель

Дать возможность в редакторе карт выделить прямоугольный параллелепипед между двумя
указанными ячейками и копировать / вырезать / удалять / вставлять его содержимое.

## UX

### Выделение

- Новый инструмент **SELECT** в хотбаре, клавиша **5** (после CUBE/WEDGE/HOST SPAWN/GUEST SPAWN).
- Дублирующий хоткей **B**: отмечает угол под прицелом, не меняя активный инструмент.
- Первый ЛКМ (в инструменте SELECT) или B фиксирует **угол 1**: берётся ячейка блока под
  прицелом (remove-ячейка из `pick()`); если под прицелом пустота/пол — ячейка установки
  (place-ячейка).
- После фиксации угла 1 полупрозрачный бокс вживую растягивается от угла 1 до ячейки под
  прицелом.
- Второй ЛКМ/B фиксирует **угол 2** — регион зафиксирован и подсвечен. Углы допустимы в любом
  порядке (нормализация min/max по каждой оси).
- Третий ЛКМ/B начинает новое выделение (угол 1 заново).
- ПКМ в инструменте SELECT сбрасывает выделение.

### Операции над выделением

Работают при любом активном инструменте, пока регион зафиксирован:

- **C** — копировать: в клипборд попадают только непустые воксели региона со всеми
  атрибутами (`t`, `c`, `d`, `f`, `bb`, `tr`, `ps`).
- **X** — вырезать: копировать + стереть регион.
- **Delete** — удалить все блоки региона (клипборд не трогается).

Спавны в выделение не входят: их ровно два и они не воксели.

### Вставка

- **V** (при непустом клипборде) включает режим вставки.
- Ghost всего фрагмента (полупрозрачные копии блоков) следует за ячейкой под прицелом;
  якорь — мин-угол фрагмента совмещается с place-ячейкой.
- **R** в режиме вставки вращает фрагмент на 90° вокруг вертикали (у клиньев пересчитывается
  направление `d`; flip `f` не меняется).
- Валидация: если хотя бы одна непустая ячейка фрагмента пересекается с существующим
  вокселем ЛИБО выходит за границы арены (те же границы, что при обычной установке блока) —
  ghost тонируется красным и ЛКМ не срабатывает.
- ЛКМ ставит фрагмент и **остаётся** в режиме вставки (серийная штамповка).
- ПКМ или смена инструмента (1–5) выходит из режима вставки.
- В режиме вставки хоткеи выделения (B, C, X, Delete) игнорируются; повторный V — no-op.
- Hint-строка редактора пополняется подсказками: 5/B выделение, C/X/Del, V вставка, R поворот.

### Вне объёма

- Undo/redo — в редакторе его нет вовсе; страховка прежняя: SAVE явный, без сохранения
  можно перезайти в карту.
- Сохранение клипборда между картами/сессиями — нет, клипборд живёт в памяти сессии
  редактора одной карты.
- Поворот вокруг горизонтальных осей, отражение — нет.

## Архитектура

Паттерн проекта: чистая логика отдельно (тестируется в vitest), сцена — тонкая.

### Новый модуль `src/editor/editorSelection.ts` (без React/THREE)

```ts
interface Fragment {
  size: [number, number, number]          // габариты bbox фрагмента в ячейках
  cells: Map<string, Cell>                // относительные координаты от (0,0,0) = мин-угла
}

regionBounds(a: Vec3i, b: Vec3i): { min: Vec3i; max: Vec3i }   // углы в любом порядке
extractRegion(voxels: Map<string, Cell>, a: Vec3i, b: Vec3i): Fragment  // только непустые
rotateFragment(frag: Fragment): Fragment                        // 90° вокруг Y; d' = (d+1)&3
canStamp(voxels: Map<string, Cell>, frag: Fragment, anchor: Vec3i, bounds: ArenaBounds): boolean
stampFragment(voxels: Map<string, Cell>, frag: Fragment, anchor: Vec3i): Map<string, Cell>
eraseRegion(voxels: Map<string, Cell>, a: Vec3i, b: Vec3i): Map<string, Cell>
```

Все функции возвращают новые `Map` (иммутабельно, как в `editorStore.ts`). Точные имена и
формы типов уточняются по месту (`Cell`, `cellKey` реэкспортируются/импортируются из
`editorStore.ts`; границы арены — те же правила, что у текущей установки блока в
`EditorScene.tsx`).

### Обвязка

- `MapEditor.tsx`: состояние `selection: { a: Vec3i; b?: Vec3i } | null`,
  `clipboard: Fragment | null`, `pasteMode: boolean`; хоткеи C/X/Delete/V/B; инструмент
  SELECT в `TOOLS` (клавиша 5).
- `EditorScene.tsx`: визуал (бокс выделения с рёбрами, маркер первого угла, ghost фрагмента
  с красным тинтом при невалидной позиции) и маршрутизация ЛКМ/ПКМ в режимах select/paste.
  Меши подсветки/ghost не участвуют в рейкасте (не получают `userData.editorTarget`).
- Никаких изменений в формате карты (`MapData`), dev-мосте и игровой части.

## Тесты

`tests/unit/editorSelection.test.ts`:

- нормализация углов: extract с углами в любом порядке даёт один и тот же фрагмент;
- extract берёт только непустые ячейки, атрибуты переносятся полностью;
- rotate: 4 поворота = identity; маппинг `d` клиньев на одном повороте корректен;
  размеры bbox меняются местами (nx↔nz);
- canStamp: false при пересечении хотя бы одной ячейки и при выходе за границы, true на
  свободном месте;
- stamp: воксели появляются по якорю; erase: регион очищен, соседи не тронуты;
- cut-сценарий: extract + erase, затем stamp в другое место — round-trip содержимого.

E2E не добавляем: редактор по проектному паттерну покрыт юнитами, pointer-lock-взаимодействие
в e2e не гоняется.
