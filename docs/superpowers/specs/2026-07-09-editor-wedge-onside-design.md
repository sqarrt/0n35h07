# Редактор карт: клин «на боку» (диагональная стена)

Дата: 2026-07-09 · Ветка: `feature/editor-copy-paste` (продолжение работ по редактору, от `release_1.1.0`)

## Цель

В редактор добавляется положение клина «на боку» — вертикальная диагональная стена: клин
стоит на всю высоту ячейки, его гипотенуза — вертикальная плоскость под 45° (вид сверху —
прямоугольный треугольник). Для скошенных углов, восьмиугольных комнат, диагональных проходов.

## Форма и геометрия

- Берём существующую геометрию `unitWedgeGeometry` (ramp: треугольник в плоскости (z,y),
  выдавлен вдоль X). Положение «на боку» = **roll 90° вокруг оси Z** (ось выдавливания X→Y,
  призма становится вертикальной, треугольник ложится в плоскость XZ), затем **yaw по dir**
  (выбор одного из 4 углов ячейки).
- Новой геометрии не требуется.
- Флип (`f`) для on-side бессмыслен: вертикальная призма симметрична по Y. При on-side флип
  игнорируется во всех путях (рендер, коллайдер).

## Модель данных

- `Cell` (в `editorStore.ts`) получает поле **`s: boolean`** (on-side). Требуемое поле, как
  `f`/`bb`/`tr`/`ps`; при построении ячеек ставится `s: false` по умолчанию.
- `MapBlock` (в `game/maps.ts`) получает опциональное **`side?: boolean`** (по образцу `flip`),
  чтобы JSON по-прежнему чисто вставлялся в игру.
- `shapeBlock(x,y,z,cell)` пишет `side: true`, когда `cell.s` (иначе поле отсутствует).
- `voxelize` читает `side` обратно в `cell.s` (по умолчанию `false`).
- Позиция (`pos`) и размер (`size`) клетки не меняются — поворот идёт вокруг центра ячейки, на
  локализацию ячейки при импорте (`Math.floor((pos−size)/S)`) не влияет.

## Общий поворот (DRY)

Один хелпер в `src/game/wedge.ts`, которым пользуются все пути:

```ts
/** Кватернион ориентации клина: on-side — roll 90° вокруг Z, затем yaw по dir. */
export function wedgeQuaternion(dir: number, side: boolean): THREE.Quaternion
```

- `src/game/blockGeometry.ts` (`blockGeometry`, wedge-ветка): вместо `g.rotateY(wedgeRotationY(dir))`
  применяет `g.applyQuaternion(wedgeQuaternion(dir, side))`. Merged-геометрия и производный от неё
  trimesh-коллайдер получают поворот автоматически.
- `src/editor/EditorScene.tsx`: меши клиньев (`ShapeMeshes`) и ghost клина берут ту же ориентацию
  (`mesh.quaternion` из хелпера) вместо `rotation=[0, wedgeRotationY(dir), 0]`.

Существующая `wedgeRotationY(dir)` остаётся (используется как yaw-часть внутри `wedgeQuaternion`).

## Игра: коллайдер

Коллайдер игры строится как trimesh из той же merged-геометрии (`blockGeometry` → `g.clone()` в
`bucketedBlockGeometries`). Поскольку геометрия уже с нужным поворотом — диагональная стена
коллайдит корректно без отдельного кода.

## Управление (редактор)

- Новая клавиша **G** переключает on-side режим кисти клина (состояние `wedgeSide` в `MapEditor`,
  по образцу `wedgeFlip`). Проп `wedgeSide` в `EditorScene`.
- `R` (yaw) и авто-ориентация от взгляда работают как раньше — выбирают `d`; в on-side это выбор
  из 4 углов ячейки.
- Ghost клина отображает on-side ориентацию.
- Хинт-строка (когда `tool === 'wedge'`) дополняется `G — on-side`.
- При установке клина ячейка получает `s: wedgeSide`; когда `wedgeSide`, флип не применяется.

## Коллизия в редакторе

`EditorScene` для ходьбы при стройке использует приблизительный heightfield (`cellTopAt` /
`surfaceInfo`). On-side клин трактуется как **куб**: плоский верх (`(y+1)·VOXEL`), сплошная
преграда. Огрубление (реально занята половина ячейки), приемлемо для перемещения по редактору;
в игре коллайдер точный. Правка: в `cellTopAt` условие наклонной поверхности —
`cell.t === 'wedge' && !cell.f && !cell.s`.

## Copy-paste

`rotateFragment` уже делает `d = (d+1)&3` для клиньев и сохраняет прочие поля через `{...cell}`.
Для on-side стен 90° поворот фрагмента = смена угла (yaw), `s` переносится — правок не нужно.

## Тесты

- Юнит `tests/unit/editorStore.test.ts` (рядом с существующими wedge round-trip): для on-side
  клина `toMapData` даёт `MapBlock.side === true`, `voxelize` возвращает `s === true`, `d`/цвет
  сохранены; обычный клин по-прежнему без `side` и с `s === false`.
- Игровой рендер и коллайдер (диагональная стена коллайдит и простреливается по флагам) —
  проверка руками в dev и в матче.

## Вне объёма

- «Заваленный» наклонный клин (roll вокруг оси движения) — не делаем, только диагональная стена.
- Авто-выбор угла on-side по геометрии наведения — оставляем текущую авто-ориентацию + R.
