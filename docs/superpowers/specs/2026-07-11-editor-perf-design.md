# Производительность редактора: инстансы клиньев + ленивая сетка

Дата: 2026-07-11 · Ветка: `feature/map-perf-chunks` (слоем; переключение на release заблокировано живыми правками os_test)

## Проблема

На большой карте (~30k вокселей, ~2620 клиньев) редактирование тормозит. На каждое
размещение/удаление создаётся новый `Map` → все `useMemo[voxels]` в `EditorScene` пересчитываются:
1. клинья рендерятся как 2620 отдельных `<mesh>` — React-реконсиляция 2620 элементов на каждую
   правку + 2620 draw-call + дорогой рейкаст по 2620 объектам;
2. edge-геометрия всех ячеек (`cellsGridGeometry`) строится **всегда**, даже когда сетка скрыта.

## Решение (лёгкий объём)

### Инстансы клиньев

Заменить компонент `ShapeMeshes` на хук `useWedgeMeshes(voxels, wedgeGeo, wedgeGeoFlip)` по образцу
`useCubeMeshes` — до 4 `InstancedMesh`: геометрия (обычная / flip) × прозрачность (opaque /
transparent). Для каждого клина инстанс несёт матрицу `compose(центр ячейки, wedgeQuaternion(dir,
side), scale = VOXEL)` и цвет (`setColorAt`).

Группировка: flip-геометрия только при `!cell.s && cell.f`; on-side (`cell.s`) — обычная геометрия
+ поворот через кватернион. Индекс группы = `(useFlip ? 2 : 0) + (cell.tr ? 1 : 0)`.

Меши тегируются `userData.editorTarget = true`. Общие геометрии `wedgeGeo`/`wedgeGeoFlip` НЕ
диспозятся при пересборке (они создаются один раз в `EditorScene`); при пересборке диспозится
только материал каждого инстанс-меша. Рендер через `<primitive>`.

### Рейкаст клиньев

Сейчас `pick()` берёт `hit.object.userData.cellKey`. У инстанс-меша ячейку даёт `hit.instanceId`,
поэтому каждый инстанс-меш клиньев несёт `userData.wedgeCellKeys: string[]` (instanceId → ключ
ячейки). В `pick()`: `const key = obj.userData.wedgeCellKeys ? obj.userData.wedgeCellKeys[hit.instanceId] : obj.userData.cellKey`.
Кубы (инстанс-меш без `wedgeCellKeys`) рейкастятся геометрически, как сейчас.

### Ленивая сетка

`useEdgesGeometry(voxels, enabled)` строит edge-геометрию только при `enabled` (`showCubeGrid`);
выключена — возвращает `null`, ни сборки, ни рендера. Рендер `<lineSegments>` — только когда
геометрия есть (убрать `visible={showCubeGrid}` — теперь заведует наличие геометрии).

## Вне объёма

- InstancedMesh кубов оставляем с полной пересборкой на правку (O(N), но без React-части — дёшево).
- Чанки/инкрементальная пересборка редактора — если после замера лёгкого варианта не хватит.

## Тесты

Изменения — three-императив в `EditorScene.tsx`, которую по проектному паттерну юнитами не
покрывают (jsdom без WebGL/r3f). Новых юнит-тестов нет; корректность — руками в dev: клинья
рисуются/ориентированы верно (обычные, flip, on-side, прозрачные), ставятся/удаляются по прицелу,
сетка (L) строится только включённой. Существующие тесты не затрагиваются. tsc/lint — агент.

## Процесс

Ветка `feature/map-perf-chunks`; тесты запускает пользователь; агент — tsc/lint. Файлы
`src/maps/os_test/*` (живые правки пользователя) не трогаю.
