# Периметр карты: стены наружу пола (сетка тютелька-в-тютельку, без потери блоков)

Дата: 2026-07-11 · Ветка: `fix/perimeter-grid` (от `release_1.1.0`)

## Проблема (две связанные)

1. **Границы не совпадают с сеткой.** `perimeter()` (`src/game/maps.ts`) ставит стену центром **на
   краю арены** (`pos z = ±hz`, полу-толщина `WALL_T = 0.25`), поэтому её внутренняя грань на
   `±(hz−0.25)` — на полклетки внутри последнего ряда ячеек. Блоки на границе пересекают стену.
2. **Пропадают блоки на границе.** `isPerimeterTrim` (`src/editor/editorStore.ts`) выкидывает при
   импорте (`voxelize`) длинные (≥8 клеток) тонкие (1 клетка) блоки, у которых ≥25% объёма в стене.
   Из-за (1) длинная тонкая полоса вдоль границы перекрывается со стеной ~на 50% → попадает под
   сигнатуру и **молча удаляется при переоткрытии карты**. (Проверено на числах.)

Обе — из-за того, что стена оседлала край вместо того, чтобы стоять снаружи.

## Решение

Сдвинуть периметр **целиком наружу** пола: центр на `±(half + WALL_T)`, внутренняя грань ровно на
`±half` (узел сетки). Размеры стен не меняем — поведение на углах как сейчас. Тогда:
- крайний ряд ячеек упирается в стену без пересечения (сетка тютелька-в-тютельку);
- легитимные блоки больше не перекрываются со стеной → `isPerimeterTrim` на них не срабатывает
  (вопрос 2 уходит следствием). Саму `isPerimeterTrim` не трогаем (она — защита от легаси
  «задвоенной стены», на чистых картах и так молчит).

### Изменения

- `src/game/maps.ts` — `perimeter()`: `pos` каждой стены сдвинуть наружу на `WALL_T`
  (N/S: `z = ±(hz+WALL_T)`; W/E: `x = ±(hx+WALL_T)`); `size` без изменений.
- `src/editor/EditorScene.tsx` — стены-меши редактора (рисуются из `half` вживую): позиции сдвинуть
  наружу на `WALL_HALF` (= `WALL_T`), размеры без изменений — чтобы совпадали с игрой.
- Регенерация **raw.json + geo.json всех карт** через dev-эндпоинт `/__recompile` (обновлён):
  хирургически заменить блоки `perimeter:true` на новые (реальным `perimeter()` через
  `ssrLoadModule`), cover-блоки не трогать (без `voxelize` → без риска потери), затем `compileBlocks`
  → geo. Запускаю headless, коммичу карты.

### Не трогаем

- Невидимые коллайдеры-границы движения в `Arena` (`BOUND_*`) — это про перемещение игрока, не про
  блоки/сетку; отдельная тема, геймплей не меняем.
- `isPerimeterTrim` — оставляем (сдвиг стены снимает ложные срабатывания на легитимных блоках).
- Уже потерянные ранее блоки не восстановить — фикс предотвращает будущие потери.

## Тесты

- Юнит: `perimeter()` — внутренняя грань каждой стены ровно на `±half` (на узле сетки), стена целиком
  вне пола (`tests/unit/maps.test.ts` или новый). Проверка: для стены N `pos.z + size.z === -half[1]`
  (внутренняя грань), `pos.z < -half[1]` (центр снаружи).
- Ручная (пользователь): в игре и редакторе стены стоят ровно по краю, крайние блоки не утоплены в
  стену; длинная полоса вдоль границы не исчезает при переоткрытии карты.

## Процесс

Ветка `fix/perimeter-grid`; агент делает автономно: tsc/lint, регенерация карт через headless
dev-сервер, коммиты; мерж в `release_1.1.0` по завершении. Тесты гоняет пользователь.
