# BASE — Базовый сценарий тестирования OneShot

Используется для самопроверки через playwright-mcp в одном и том же открытом окне браузера.

---

## Подготовка

1. Убедиться что `npm run dev` запущен (`http://localhost:5173`)
2. Открыть страницу через `browser_navigate` (один раз за сессию)
3. Сделать скриншот — убедиться что показан экран "ONESHOT / Click to play"

---

## Сценарии

### 1. Стартовый экран
- **Ожидается:** оверлей с текстом "ONESHOT", "Click to play", подсказками управления
- **Проверка:** скриншот, наличие текста в snapshot

### 2. Рендер сцены
- **Ожидается:** 3D-арена видна за оверлеем: пол с сеткой, стены, оранжевая мишень
- **Проверка:** скриншот — видна геометрия и мишень

### 3. HUD расположение
- **Ожидается:**
  - Прицел `+` строго по центру экрана (50% / 50%)
  - Бары `BEAM [ЛКМ]` и `SHIELD [ПКМ]` внизу по центру, фиксированные
- **Проверка:** скриншот после снятия оверлея (pointer lock); элементы не смещаются при движении

### 4. Консоль — нет ошибок
- **Ожидается:** 0 errors в console
- **Проверка:** `Console: 0 errors` в выводе `browser_take_screenshot`

### 5. Cooldown-бары заполнены
- **Ожидается:** обе полоски полностью заполнены при старте (готовы)
- **Проверка:** скриншот — бары cyan и blue полной ширины

---

## После каждого изменения кода

1. Обновить страницу через `browser_navigate` на `http://localhost:5173`
2. Сделать скриншот
3. Проверить: 0 errors, HUD на месте, сцена рендерится
4. Если есть ошибки — прочитать console log и исправить

---

## Важно: pointer lock в playwright

`requestPointerLock()` требует реального пользовательского жеста — playwright не может его вызвать автоматически. Для тестирования механик (стрельба, щит) использовать мок:

```js
Object.defineProperty(document, 'pointerLockElement', {
  get: () => document.body, configurable: true
})
// ... тест ...
Object.defineProperty(document, 'pointerLockElement', {
  get: () => null, configurable: true
})
```

Позиция камеры доступна через `window.__debugCamera.position` (debug-хелпер в Game.tsx).
